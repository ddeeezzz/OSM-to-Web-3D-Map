# osm_to_web_3d_map Engine

本项目起源于**西南交通大学犀浦校区**的一次数据可视化实践。在完成校园建模的过程中，抽象出了一套通用的数据转换逻辑，支持通过输入标准的 OpenStreetMap (OSM) 原始数据，自动化生成具备基础交互能力的 3D 地图场景。

## 核心实现能力

本项目通过对西南交大校园要素的建模，实现了以下自动化流程：

1. **自动化几何重构**: 自动解析 OSM 标签，利用 WebGL 实时拉伸平面轮廓，将 2D 地图数据转化为具备物理高度的 3D 建筑群。
2. **多层级拓扑渲染**: 采用 **Three.js** 核心渲染架构，能够同时处理数万个地理要素（建筑、植被、道路、水体）的像素级渲染与交互。
3. **动态路径规划**: 自动提取 OSM 中的道路拓扑网络，内置基于图论的搜索算法，支持实时步行/交通导航及动态时间预估。
4. **无缝交互集成**: 为生成的每一个地理实体自动绑定点击事件与悬停反馈，支持模糊搜索与 POI 信息流展示。

## 数据更换指南 (OSM to 3D-Map)

如果你希望将本项目应用于其他区域，请遵循以下步骤与注意事项：

### 1. 准备数据
- 将目标区域的 OSM 数据导出并命名为 `map.osm`，放置在项目根目录。
- **注意事项**: 
    - **自定义高度映射**: 若原始数据缺失 `building:levels` 或 `height` 标签，可通过修改 `app/src/config/index.js` 中的 `heights` 配置项，按建筑类型（如 `university`, `dormitory`, `office`）手动赋予全局基准高度。
    - 建议导出范围不宜过大（如超过 5km x 5km），以免影响 WebGL 渲染性能。

### 2. 执行预处理流水线
必须依次执行以下脚本以完成数据转换与清洗：
```bash
# 1. 基础转换 (OSM -> GeoJSON)
node tools/convert-osm.js

# 2. 数据清洗与几何优化
node tools/clean-geojson.js

# 3. 兴趣点 (POI) 提取
node tools/extract-poi.js

# 4. 构建导航路网图
node tools/build-road-graph.js
```

### 3. 配置对齐 (可选)
- 如果新区域的经纬度跨度较大，可能需要调整 `app/src/config/index.js` 中的初始视角（Camera View）与坐标偏移量，以获得最佳显示效果。


- **数据驱动生成**: 仅需提供 `map.osm` 文件，引擎即可自动化解析并渲染建筑、道路、水体、绿地等地理要素。
- **高阶渲染架构**: 基于 **Three.js** 核心渲染技术，支持大规模地理空间数据的高效 WebGL 展示。
- **智能路径规划**: 内置基于地理拓扑结构的路径搜索算法，支持多模式（步行、骑行、驾车）路径规划与时间预估。
- **响应式 UI 系统**: 采用 **React 19** + **Zustand** 构建的现代化交互界面，支持模糊搜索、POI 定位与详细信息展示。
- **地理数据工具链**: 内置 `tools/` 脚本库，支持 OSM 到 GeoJSON 的深度清洗与数据格式化。


### 前置环境要求

- **Node.js**: v16 及以上版本 (建议使用 LTS 版本)
- **pnpm**: 包管理工具 (`npm install -g pnpm`)

### 1. 准备数据

将标准的 OSM 数据文件重命名为 `map.osm` 并放置在项目根目录。

### 2. 安装与运行


```bash
# 进入应用目录
cd app

# 安装依赖
pnpm install

# 启动开发服务器
pnpm run dev

```
之后，在浏览器中访问 `http://localhost:5173` 即可看到应用。

## 技术栈

- **渲染内核**: Three.js (3D 场景控制)
- **前端框架**: React 19 + Vite
- **状态管理**: Zustand
- **地理处理**: osmtogeojson, turf.js
